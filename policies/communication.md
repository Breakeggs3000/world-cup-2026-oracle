# Communication policy

## With the user

- Lead with the answer or outcome; add detail as needed.
- State assumptions explicitly when making them.
- Ask one focused question when blocked — not a list of ten.
- Report what you tried when something fails.

## In artifacts and docs

- Use clear headings and short paragraphs.
- Link to files with full paths when referencing the repo.
- Separate facts from recommendations.

## Between agents (handoffs)

When passing work in a crew:

1. Summarize what was done and what remains.
2. List files touched or created.
3. Include acceptance criteria status.
4. Note risks and open questions.

Handoff files live in `workspace/artifacts/` unless the task spec says otherwise.
